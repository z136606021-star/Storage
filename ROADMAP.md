- [x] Flyway 接管运行时 schema，增量升级保留已有卷；测试数据与运行时业务数据隔离。
- [x] Shiro + JWT Bearer token、Pinia 登录态、动态菜单和动态路由。
- [x] 按 `common/system/warehouse/design/experience/infrastructure` 业务域组织后端单模块。
- [x] Vue 页面、API 和类型按业务域组织；路由组件通过 `import.meta.glob` 解析菜单 `component_key`。
- [x] Less 公共样式、公共 CRUD 组件/composable 和仓库筛选/表格复用层。
- [x] 仓库、系统和设计域导出复用 EasyExcel 公共写出层；经验库保留 AutoPoi；复杂导入保留 POI + `ExcelCellUtils` + 行级错误契约。
- [x] MinIO 通用上传、BOM 多图片、上传上限 SSOT 和 Nginx 运行时渲染。
- [x] 请求 ID、未预期异常落库、前端异常上报和 30 天清理策略。

## 发布前剩余工作

以下事项是发布门禁或目标环境验收，不代表继续扩展业务范围：

- [ ] 使用目标环境 `.env` 启动 `docker-compose.yml`，确认后端健康检查通过且前端入口可访问。
- [ ] 备份现有 MySQL，并验证 Flyway 从目标库当前版本增量升级到最新版本。
- [ ] 使用真实业务样例验收完整链路：配置 Bin/BOM -> 入库 -> 台账 -> 出库 -> 安全库存 -> 采购清单 -> 统计。
- [ ] 验证目标 MinIO 的上传、预览、下载和 BOM 多图片流程。
- [ ] 验证目标 SMTP 的注册验证码、登录态验证码改密和忘记密码链接。
- [ ] 轮换管理员、MySQL、MinIO、SMTP 和 JWT 默认凭据，关闭不适合生产的 admin 自动重置能力。
- [ ] 在 HTTPS/网关侧确认公网入口、限流、请求体上限和跨域策略。
- [ ] 执行后端测试、前端测试/构建及关键 API 非法 body/库存不变量回归。
- [ ] 维护者逐文件审查发布 diff，并在健康检查通过后创建首个正式语义化版本 tag。

## 非阻塞优化

- [ ] 根据真实使用反馈补充仓库端到端浏览器测试，优先覆盖批量出入库和 Excel 导入。
- [ ] 为 MySQL/MinIO/异常日志配置生产监控、告警和周期性恢复演练。
- [ ] 多实例部署前将进程内限流和 JWT 撤销状态迁移到共享存储或网关。
- [ ] 持续清理根目录 API/type 兼容 shim；仅在所有旧 import 消失后删除。

## 后续模块

设计指引和经验库已经形成独立业务域及基础 CRUD/Excel/附件能力，但 `V020__retain_warehouse_and_system_navigation.sql` 将其默认菜单隐藏。后续恢复或扩展时必须单独确认：

- 产品范围、角色权限、验收数据和是否正式开放菜单。
- 与项目主数据的关联方式；当前不得反向耦合仓库库存不变量。
- 是否需要补充审批、版本、审计、搜索或报表能力。
- 对应迁移、测试、README 功能说明和 CHANGELOG 记录。

项目中心、采购审批、财务结算、客户验收和项目进度编排仍未纳入本仓库模块，不能为了“平台完整”直接塞入仓库域。

## 维护规则

- 当前用户可见功能、启动方式或部署配置变化时更新 `README.md`。
- 阶段性落地事项记录到 `CHANGELOG.md`。
- 只有长期规则、架构边界或质量门禁变化时更新 `AGENTS.md`。
- 路线图事项完成后立即勾选；新增业务方向先确认边界和验收标准，再进入实现计划。
